- 영화 프로젝트 RDB 설계

## 학습한 내용

- 1:N, M:N 관계 맺어주기
  - model 변경후 꼭 migration해주기
  - 같은 테이블에서 관계 맺어줄 때는, 'self' 사용, symmetrical=False 해주기
- 관계 맺은 테이블의 데이터 참조
  - related_name 설정으로 참조 모델 이름 변경 가능
    <br>
    (특히 한 테이블에서 같은 테이블 여러번 관계 맺을 때 설정해줘야 함)
- 부트스트랩을 이용해서 css 먹이기
